# IP 주소 체계

## IPv4 주소 구조

```
32bit = 4 옥텟 (8bit × 4)

192  .  168  .   1   .  100
11000000.10101000.00000001.01100100

표현: 10진수 점 표기법 (Dotted Decimal Notation)
범위: 0.0.0.0 ~ 255.255.255.255 (약 43억 개)
```

### 네트워크 부분 vs 호스트 부분

```
IP 주소 = 네트워크 주소 + 호스트 주소

192.168.1.100 / 24
              └── 서브넷 마스크: /24 = 255.255.255.0

네트워크: 192.168.1.0  (앞 24bit)
호스트:   .100         (뒤 8bit → 256-2=254개 사용 가능)
```

## 사설 IP vs 공인 IP

### 사설 IP (Private IP, RFC 1918)
인터넷에서 라우팅되지 않음. 내부 네트워크에서만 사용.

```
클래스 A: 10.0.0.0    ~ 10.255.255.255   (/8)   → 1677만 개 호스트
클래스 B: 172.16.0.0  ~ 172.31.255.255   (/12)  → 104만 개 호스트
클래스 C: 192.168.0.0 ~ 192.168.255.255  (/16)  → 65534개 호스트
```

### 특수 IP 주소
```
127.0.0.0/8    → Loopback (127.0.0.1 = localhost)
169.254.0.0/16 → Link-Local (APIPA: DHCP 실패 시 자동 할당)
0.0.0.0        → "모든 IP", 기본 라우트 (default route)
255.255.255.255 → 제한적 브로드캐스트
[네트워크주소].255 → 해당 네트워크 브로드캐스트 (예: 192.168.1.255)
```

---

## CIDR (Classless Inter-Domain Routing)

과거 클래스 기반(A/B/C)을 대체. 유연한 서브넷 크기 지원.

```
표기법: IP주소/프리픽스길이
예시:
  10.0.0.0/8    → 1677만 호스트 (기업 내부망)
  172.16.0.0/16 → 65534 호스트
  192.168.1.0/24 → 254 호스트 (가정/소규모 오피스)
  10.0.1.0/28   → 14 호스트 (소규모 서브넷)
  10.0.0.1/32   → 단일 호스트 (라우팅 규칙에서 특정 IP 지정)
  0.0.0.0/0     → 모든 IP (기본 게이트웨이 경로)
```

## AWS VPC 주소 설계 예시

```
VPC: 10.0.0.0/16 (65534개 IP)
│
├── Public Subnet (AZ-a):  10.0.0.0/24   (254 IP) ← ALB, Bastion
├── Public Subnet (AZ-b):  10.0.1.0/24   (254 IP)
├── Private Subnet (AZ-a): 10.0.10.0/24  (254 IP) ← App 서버
├── Private Subnet (AZ-b): 10.0.11.0/24  (254 IP)
├── DB Subnet (AZ-a):      10.0.20.0/24  (254 IP) ← RDS
└── DB Subnet (AZ-b):      10.0.21.0/24  (254 IP)
```

## IPv6 기초

```
128bit = 16진수 8그룹 × 16bit
예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

축약 규칙:
1. 각 그룹 앞자리 0 생략: 0db8 → db8
2. 연속된 0 그룹 → ::  (한 번만 사용 가능)
   2001:db8:85a3::8a2e:370:7334

특수 주소:
  ::1          → Loopback (IPv4의 127.0.0.1)
  fe80::/10    → Link-Local
  fc00::/7     → Unique Local (IPv4 사설 IP 역할)
  2000::/3     → 공인 IPv6 (글로벌 유니캐스트)
```

### IPv4 vs IPv6 비교
| 항목 | IPv4 | IPv6 |
|------|------|------|
| 주소 크기 | 32bit (43억) | 128bit (340언더슈틸리온) |
| 헤더 크기 | 20~60 byte | 40 byte (고정) |
| 단편화 | 라우터에서 수행 | 송신 측에서만 수행 |
| ARP | ARP 프로토콜 | NDP (Neighbor Discovery Protocol) |
| 브로드캐스트 | 있음 | 없음 (멀티캐스트로 대체) |
| NAT | 필수 | 불필요 (주소 충분) |
| 헤더 체크섬 | 있음 | 없음 (L4에서 처리) |

## 면접 단골 질문

**Q. 사설 IP와 공인 IP의 차이는?**
> 사설 IP는 RFC 1918에 정의된 내부 전용 주소 (10.x.x.x, 172.16-31.x.x, 192.168.x.x). 인터넷으로 직접 라우팅 불가 → NAT 필요. 공인 IP는 인터넷에서 유일하게 라우팅 가능한 주소.

**Q. 169.254.x.x IP가 할당됐을 때 무엇을 의심해야 하나요?**
> DHCP 서버에서 IP를 받지 못한 것. APIPA(Automatic Private IP Addressing)로 자동 할당된 것. DHCP 서버 연결, 네트워크 케이블, DHCP 설정 확인 필요.

**Q. /24와 /25 서브넷의 차이는?**
> /24 → 호스트 8bit → 256-2=254 사용 가능. /25 → 호스트 7bit → 128-2=126 사용 가능. /25는 /24를 반으로 나눈 것.
