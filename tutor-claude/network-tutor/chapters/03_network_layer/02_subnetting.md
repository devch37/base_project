# 서브네팅 (Subnetting)

## 핵심 공식

```
서브넷 마스크 /N:
  네트워크 비트: N개
  호스트 비트:   32-N개
  호스트 수:     2^(32-N) - 2  (네트워크 주소 + 브로드캐스트 제외)
  서브넷 수:     2^(빌린 비트 수)
```

---

## 프리픽스 길이 → 빠른 계산표

| 프리픽스 | 서브넷 마스크     | 호스트 수 | 활용 사례 |
|---------|----------------|---------|---------|
| /8      | 255.0.0.0      | 16,777,214 | 대형 기업 내부망 |
| /16     | 255.255.0.0    | 65,534   | VPC 전체 |
| /24     | 255.255.255.0  | 254      | 일반 서브넷 |
| /25     | 255.255.255.128| 126      | /24를 반분 |
| /26     | 255.255.255.192| 62       | 소규모 |
| /27     | 255.255.255.224| 30       | 매우 소규모 |
| /28     | 255.255.255.240| 14       | 최소 서브넷 |
| /30     | 255.255.255.252| 2        | P2P 링크 |
| /31     | 255.255.255.254| 2        | P2P (네트워크/브로드캐스트 없음) |
| /32     | 255.255.255.255| 1        | 단일 호스트 |

## 서브네팅 계산 예시

### 문제: 192.168.10.0/24를 4개의 동일한 서브넷으로 분리

```
필요한 서브넷: 4개 → 2^2 = 4 → 2비트 빌림
원래: /24 → 새 마스크: /26

각 서브넷:
  192.168.10.0/26   → .0   ~ .63   (호스트: .1 ~ .62)
  192.168.10.64/26  → .64  ~ .127  (호스트: .65 ~ .126)
  192.168.10.128/26 → .128 ~ .191  (호스트: .129 ~ .190)
  192.168.10.192/26 → .192 ~ .255  (호스트: .193 ~ .254)

각 서브넷당 호스트: 2^6 - 2 = 62개
```

### 계산 방법 (손으로)

```
/26 서브넷 마스크: 255.255.255.192
  → 마지막 옥텟: 11000000 = 192
  → 호스트 비트: 00111111 = 63
  → 블록 크기: 64 (= 256 - 192)

서브넷 시작 주소: 0, 64, 128, 192 (블록 크기 = 64씩 증가)
```

## VLSM (Variable Length Subnet Mask)

다양한 크기의 서브넷을 혼합 사용:

```
회사 네트워크: 172.16.0.0/16

부서별 요구사항:
  본사 (500 호스트 필요)  → 172.16.0.0/23  (510 호스트)
  지사 A (100 호스트)    → 172.16.2.0/25  (126 호스트)
  지사 B (50 호스트)     → 172.16.2.128/26 (62 호스트)
  서버팜 (10 호스트)     → 172.16.2.192/28 (14 호스트)
  P2P 링크 (2 호스트)    → 172.16.2.208/30 (2 호스트)

→ IP 낭비 최소화
```

## AWS 서브넷 설계 실전

```
VPC: 10.0.0.0/16

설계 원칙:
1. AZ(가용 영역)별 대칭 구조
2. Public/Private/DB 계층 분리
3. 미래 확장을 위해 넉넉하게 할당

10.0.0.0/16 분할:
  ┌─ 10.0.0.0/18   → AZ-a 전체 (16384 IP)
  │    ├─ 10.0.0.0/24   → Public (254)
  │    ├─ 10.0.10.0/24  → Private App (254)
  │    └─ 10.0.20.0/24  → Private DB (254)
  │    (나머지는 예약)
  │
  ├─ 10.0.64.0/18  → AZ-b 전체
  │    ├─ 10.0.64.0/24  → Public
  │    ├─ 10.0.74.0/24  → Private App
  │    └─ 10.0.84.0/24  → Private DB
  │
  └─ 10.0.128.0/17 → 미래 확장용 예약
```

## 서브넷 마스크 계산 빠른 팁

```
"내가 속한 네트워크 주소는 무엇인가?"
  IP: 192.168.1.100
  Mask: /26 (255.255.255.192)

  계산: IP AND Mask
  100   = 01100100
  192   = 11000000
  AND   = 01000000 = 64

  → 네트워크 주소: 192.168.1.64
  → 브로드캐스트:  192.168.1.127 (.64 + 63)
  → 사용 가능:     .65 ~ .126
```

## 실습 과제

```bash
# 1. 현재 IP와 서브넷 확인
ifconfig en0
ip addr show eth0
# inet 192.168.x.x/24 → /24가 서브넷 마스크

# 2. 서브넷 계산기 (Python 한 줄)
python3 -c "import ipaddress; n=ipaddress.ip_network('192.168.1.0/26'); print(f'호스트 수: {n.num_addresses-2}, 범위: {list(n.hosts())[0]} ~ {list(n.hosts())[-1]}')"

# 3. 서브넷 쪼개기
python3 -c "
import ipaddress
net = ipaddress.ip_network('192.168.10.0/24')
for subnet in net.subnets(new_prefix=26):
    hosts = list(subnet.hosts())
    print(f'{subnet}: {hosts[0]} ~ {hosts[-1]} ({len(hosts)} hosts)')
"
```

## 면접 단골 질문

**Q. /24 네트워크에서 호스트 수가 왜 254개인가요?**
> 호스트 비트 8개 → 2^8=256. 첫 주소(네트워크 주소, 예: .0)와 마지막 주소(브로드캐스트, 예: .255)는 호스트에 할당 불가 → 256-2=254.

**Q. AWS VPC에서 각 서브넷이 실제 사용 가능한 IP가 5개 적은 이유는?**
> AWS가 처음 4개 + 마지막 1개를 예약: .0 네트워크, .1 VPC 라우터, .2 DNS, .3 예약, .255 브로드캐스트. /24 서브넷 → 256-5=251개 사용 가능.
