# 07. 인프라/클라우드 보안 기초

## 목표
- 배포 환경에서 발생하는 보안 위험을 이해한다.
- 최소 권한과 네트워크 분리를 설계할 수 있다.

## 핵심 개념
- 네트워크 분리, 방화벽/보안그룹
- IAM/권한 설계
- 비밀 관리(Secrets) 원칙

## 설명(주석)
개발자가 보안 사고를 “코드”로만 생각하면 위험하다. 실제 사고는 **설정 실수**나 **권한 과다**에서 자주 시작된다.

## 체크리스트
- 서비스가 필요한 포트/권한만 열어두었는가
- 개발용 계정과 운영 계정을 분리했는가
- 시크릿이 코드/리포지토리에 포함되지 않는가
