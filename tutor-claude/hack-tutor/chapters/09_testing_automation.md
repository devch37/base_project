# 09. 보안 테스트와 자동화

## 목표
- 보안 검사를 개발 파이프라인에 통합한다.
- 도구의 한계를 이해하고 보완한다.

## 핵심 개념
- SAST/DAST 개념과 한계
- 의존성 취약점 관리
- SBOM 개념

## 설명(주석)
자동화는 **빨리 발견하고, 빨리 고치는** 능력을 키운다. 다만 도구는 오탐/미탐이 있으므로 **리뷰와 정책**이 함께 가야 한다.

## 체크리스트
- 빌드 파이프라인에 기본적인 취약점 스캔이 있는가
- 의존성 업데이트 정책이 있는가
- 중요한 경로에 보안 테스트가 있는가
