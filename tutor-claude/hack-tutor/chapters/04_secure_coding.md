# 04. 안전한 코딩 패턴 (Java/Spring)

## 목표
- Java/Spring에서 흔한 보안 실수를 예방하는 패턴을 익힌다.
- 코드 리뷰 체크리스트를 만든다.

## 핵심 개념
- 입력 검증과 데이터 정규화
- 파라미터 바인딩/ORM 사용 원칙
- 파일 업로드/다운로드 보안
- 예외 처리와 에러 메시지 최소화

## 설명(주석)
보안은 “특정 취약점 대응”보다 **안전한 기본 패턴**을 확립하는 것이 효율적이다. Java/Spring에서는 프레임워크가 제공하는 안전 장치를 제대로 사용하는지가 관건이다.

## 체크리스트
- 컨트롤러에서 직접 SQL 문자열을 조합하지 않는가
- DTO/검증 애너테이션을 통해 입력을 검증하는가
- 업로드 파일의 타입/크기/저장 경로를 검증하는가
